# MVPベンチマーク

MVP計画（[03-plan/01-mvp-plan.md](../03-plan/01-mvp-plan.md) §5）で定めた公開KPIの計測記録。数値は計測日・環境つきで更新していく。

## 計測結果（2026-07-17時点）

| KPI                               | 目標   | 実測                                                                | 環境・条件                                            |
| --------------------------------- | ------ | ------------------------------------------------------------------- | ----------------------------------------------------- |
| 導入時間（init→最初のテスト成功） | ≤ 5分  | **19〜24秒**（flutter create込み）                                  | macOS / iOSシミュレータ / ビルドキャッシュ温          |
| イテレーション（修正→再実行完了） | ≤ 10秒 | **テスト実時間 + 約0.5秒**（5テスト8.9秒のスイートで9.4秒）         | `endevir develop` / iOSシミュレータ                   |
| CIフレーク率                      | < 1%   | **0%**（ローカルiOS: 50/50連続、GitHub Actions Android: 25/25連続） | 意図的にflakyな4画面を含む5テストスイート             |
| trace/証跡記録オーバーヘッド      | < +10% | **実質0%**（証跡モード=全ステップスクショON）                       | ADR-004の遅延エンコード。S4スパイク実測: 同期は1ms/枚 |

## スイート構成（計測対象）

examples/flutter_app の5テスト。すべてsleepなし・イベント駆動待機:

1. ホーム画面の表示検証
2. 無限アニメーション下でのタップ+検証（pumpAndSettle不能なケース)
3. 3秒の遅延ロード待機
4. 有限アニメーション（800ms）の完了検知
5. フォーム入力+擬似遅延つき送信（テキスト入力・バリデーション）

## 計測方法

- 導入時間: `flutter create` → `endevir init` → `endevir test` 成功までの実時間（`scripts/` 外で手動計測）
- イテレーション: `endevir develop` が毎ループ表示する `loop: xxxms`（コマンド内蔵計測）
- フレーク率: [scripts/soak.sh](../../scripts/soak.sh)（N回連続実行）。CIでは Device Tests ワークフローの `workflow_dispatch` で任意回数を実行可能
- 記録オーバーヘッド: S4スパイク（ADR-004）のモード比較 + 証跡モードON/OFFのスイート時間比較
